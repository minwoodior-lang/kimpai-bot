1. 전체 레이아웃 폭 정렬 (헤더까지 통일)

문제

상단 헤더(“KimpAI 로고 ~ 로그인/회원가입/라이트모드 토글”)가 아직도 화면 전체를 쓰고 있고,
메인 콘텐츠(max-w-[1200px])와 폭이 안 맞습니다.

요청

TopMarketInfoBar + Navbar 를 감싸는 컨테이너도 무조건
max-w-[1200px] mx-auto px-4 lg:px-5 로 맞춰 주세요.

즉, 페이지 전체 구조가 아래처럼 동일해야 합니다.

<body className="bg-[기존네이비색]">
  <header className="sticky top-0 z-50 bg-[네이비백그라운드]">
    <div className="max-w-[1200px] mx-auto px-4 lg:px-5">
      {/* TopMarketInfoBar + Navbar */}
    </div>
  </header>

  <main className="max-w-[1200px] mx-auto px-4 lg:px-5">
    {/* 차트, 코인 테이블 */}
  </main>

  <footer className="max-w-[1200px] mx-auto px-4 lg:px-5">
    {/* Footer */}
  </footer>
</body>

2. 바탕색 / 다크모드 기본 톤 복구

문제

전체 배경이 너무 진한 거의 블랙으로 바뀌었고,
중간에 컨텐츠를 한 번 더 감싸는 검정 박스 때문에 화면이 답답합니다.

예전 KimpAI 네이비 톤(그라디언트 느낌)이 사라졌습니다.

요청

페이지 전체 기본 배경을 예전 네이비 톤으로 복구해 주세요.

예: bg-[#020617] ~ bg-gradient-to-b from-[#020617] via-[#020617] to-[#050816]

핵심: 예전처럼 “어두운 네이비 그라데이션”으로 보이면 됩니다.

차트/테이블을 감싸는 추가 검은 박스 배경은 제거해서
컨텐츠가 바로 바탕색 위에 놓이도록 해 주세요.
(지금처럼 “위에 또 사각형이 하나 올라간 느낌”이 안 나게.)

3. 라이트 모드 진짜 구현 (배경 하얀색 + 글자색/테이블 전부)

문제

헤더에 라이트/다크 토글(☀️/🌙)은 있는데,
눌러도 배경/글자/테이블 색상이 안 바뀝니다.
사실상 아이콘만 바뀌고 테마는 동작 안 함.

요청

next-themes 기준 정석 구현:

_app.tsx 또는 layout.tsx:

import { ThemeProvider } from "next-themes";

export default function App({ Component, pageProps }: AppProps) {
  return (
    <ThemeProvider attribute="class" defaultTheme="dark">
      <Component {...pageProps} />
    </ThemeProvider>
  );
}


body 클래스에 darkMode=class 대응:

다크 모드:

<body className="bg-[#020617] text-slate-100">


라이트 모드:

className="bg-white text-slate-900 dark:bg-[#020617] dark:text-slate-100"


핵심 UI 컴포넌트들에 light/dark 색상 분리:

TodayPremiumSection 카드, PRO 락 카드

코인 테이블 thead / tbody / hover

드롭다운, 검색창, 버튼들

예시:

<div className="rounded-xl border border-slate-200 bg-white dark:border-slate-800 dark:bg-slate-900/80">


라이트 모드에서 배경은 거의 흰색, 글자는 진한 회색/검정으로
절대 글자가 안 보이는 상황이 없도록 명시적으로 색을 지정해 주세요.

4. “오늘의 AI 김프 요약” 섹션 레이아웃 재구성 (PC / 모바일)

문제

상단 “오늘의 AI 김프 요약” 섹션이 아직도 너무 휑하고,
좌측 텍스트 + 우측 카드/버튼의 레이아웃이 어색합니다.

이전에 있던 문구( PRO 전용 48시간 김프 예측 설명 )도 위치가 바뀌거나 사라졌고,
버튼 텍스트도 다시 PRO 분석으로 돌아가 있음.

TodayPremiumSection 컴포넌트 기준으로 아래처럼 구성해 주세요.

모바일

위 구조 그대로 flex-col 우선,
텍스트/카드/락 카드가 위에서 아래로 자연스럽게 쌓이게 해 주세요.

마지막 설명 문구(“최근 30일 기준…”)가 모바일에서도 줄바꿈만 되지 짤리지 않도록
text-[10px] 정도로 줄여서라도 전부 보이게 해 주세요.

5. 코인 테이블 – 아이콘/별/코인명 정렬 + 줄무늬 제거

문제

코인 셀 내부에서
아이콘 – 별 – 한글명 – 영어 심볼의 높이가 제대로 안 맞습니다.

각 행의 배경이 교차 색으로 보이거나, 라인이 어색합니다.
KIMPGA처럼 “행 배경색은 단일, 행 사이 경계선만 얕게” 가 좋습니다.

요청 – 코인 셀 구조

<td className="py-2.5 px-3">
  <div className="flex items-center gap-3">
    {/* 아이콘 + 별 세로 정렬 */}
    <div className="flex flex-col items-center justify-center gap-1">
      <CoinIcon ... className="h-7 w-7" />
      <button className="text-[11px] text-slate-500">
        {isFav ? "★" : "☆"}
      </button>
    </div>

    {/* 이름 영역 */}
    <div className="flex flex-col justify-center">
      <span className="text-sm font-medium text-slate-100 truncate">
        {name_ko}
      </span>
      <span className="text-[11px] text-slate-400 uppercase tracking-[0.02em]">
        {symbol}
      </span>
    </div>
  </div>
</td>


items-center + flex-col 로 아이콘과 별의 수직 중앙을 맞춰
코인명 두 줄과 같은 높이에 자연스럽게 맞도록 해 주세요.

요청 – 행 스타일

index % 2 로 배경색 바꾸는 로직 완전히 제거.

공통 스타일 예시:

<tr className="border-b border-slate-800 hover:bg-slate-900/60 last:border-b-0">


라이트 모드에서는:

<tr className="border-b border-slate-200 hover:bg-slate-50">

6. 모바일 코인 테이블 – 1페이지 안에 5개 컬럼 보이게

문제

모바일에서 코인 그리드가 다시 한 화면에 다 안 들어오고
가로 스크롤/잘림이 생깁니다.

요청

모바일(최소 360~390px 폭) 기준으로
아래 5개 컬럼이 가로 스크롤 없이 한 화면에 들어오게 해 주세요.

코인명

현재가

김프

전일대비

거래액(일)

구현 방법 예시:

text-[10px] ~ text-xs, px-1.5, whitespace-nowrap 적극 사용

필요하면 숫자만 남기고 “%” 등 최소 표기로 축약

header:

<th className="text-[10px] px-1.5 py-1 text-slate-400">

7. 상단 고정 바 톤 & 모바일 오버플로우

문제

모바일에서 상단 환율/테더/BTC점유율 바가
flex-wrap이 제대로 안 먹거나, 옆으로 밀리는 이슈가 있었습니다.

전체 배경 톤과 다른 진한 색이 섞여서 두 군데 색이 갈리는 느낌입니다.

요청

TopMarketInfoBar 내부 컨테이너:

<div className="flex flex-wrap items-center gap-x-4 gap-y-1 text-[11px]">


바탕색:

상단 고정 바: bg-[#020617] (조금 진한 네이비)

그 아래 메인: 동일 톤 계열 – 전체가 자연스럽게 이어져 보이게.

8. 라이트/다크 토글 위치 & 동작 확인

요청

헤더 오른쪽: 라이트/다크 토글 → 로그인 왼쪽에 위치 고정.

모바일 메뉴 열린 상태에서도 같은 토글이 메뉴 안 상단에 노출되어야 합니다.

토글 시:

body에 class="light" / class="dark" 제대로 반영

메인, 테이블, 카드 모두 색상 변경 되는지 다시 한 번 실제로 확인해 주세요.

“이미 했다”가 아니라, 현재 배포 화면 기준으로 눈으로 확인 가능한 결과가 맞는지 체크하면서 한 번에 처리 부탁드립니다.
