최종 요약 잘 봤습니다. 실제 화면 기준으로 보니 아직 3가지 문제가 남아 있어서 추가 수정 요청드립니다.

=====================================
1) 홈 상단 차트 동작 이상
=====================================
현재 보고 있는 증상은:

- 홈(/) 상단 차트가 여전히 이상하게 열립니다.
- 예전에 제가 구성해둔 차트 드롭다운/타임프레임 컨트롤이 깨진 느낌이고,
- 차트가 원하는 대로 BTC/USDT 한 개만 깔끔히 나오지 않습니다.

제가 원하는 상태는:

- 홈(/)에서는 TradingView 차트가 **단일 1개**만 렌더링
- 기본 심볼: **BINANCE:BTCUSDT**
- 나머지 타임프레임/컨트롤 UI는 제가 이전에 사용하던 형태 그대로 유지  
  (지금처럼 1시간 고정 + 버튼 비활성화가 아니라, 기존 컨트롤이 살아있는 상태에서 "초기값만 BTC/USDT"가 되면 됩니다)

즉, `ChartWithControls.tsx`에서
- 홈 라우트 전용으로 `defaultSymbol = "BINANCE:BTCUSDT"` 정도만 설정하고,
- interval/타임프레임 고정이나 버튼 비활성화는 제거해 주시면 됩니다.

혹시 현재 홈에서 쓰는 차트 컴포넌트 구조가 어떻게 되어 있는지
`ChartWithControls.tsx` 관련 부분을 한 번만 공유해 주시면, 필요한 경우 ChatGPT 쪽에서 심볼/props를 정확히 잡아드리겠습니다.

=====================================
2) 해외 거래소 드롭다운 라벨 수정
=====================================
지금 해외 거래소 드롭다운 라벨이 아래처럼 애매하게 보입니다.

- "바이낸스"
- "바이낸스"
- "바이낸스 선물"
- OKX, Bybit, Bitget, Gate, HTX, MEXC …

사용자 입장에서 어떤 마켓인지 바로 보이도록, 라벨을 아래처럼 바꿔주세요.
(내부 value는 지금 쓰시는 enum/상수 그대로 써도 되고, **보이는 텍스트만** 이렇게 맞추면 됩니다.)

- BINANCE_BTC   → "바이낸스 BTC 마켓"
- BINANCE_USDT  → "바이낸스 USDT 마켓"
- BINANCE_FUTURES (USDS-M) → "바이낸스 선물 (USDⓈ-M)"

- OKX_USDT   → "OKX USDT 마켓"
- BYBIT_USDT → "Bybit USDT 마켓"
- BITGET_USDT → "Bitget USDT 마켓"
- GATE_USDT  → "Gate USDT 마켓"
- HTX_USDT   → "HTX USDT 마켓"
- MEXC_USDT  → "MEXC USDT 마켓"

지금처럼 심볼/아이콘은 잘 보이니, 한글 라벨만 위처럼 교체해 주시면 됩니다.

=====================================
3) 빗썸/코인원 선택 시 무한 로딩 (업비트만 정상)
=====================================
현재 프리미엄 테이블에서:

- 기준 거래소를 **업비트 KRW**로 두면 코인 리스트가 정상적으로 잘 나오는데,
- 기준 거래소를 **빗썸 KRW, 빗썸 BTC, 빗썸 USDT, 코인원 KRW**로 바꾸면
  → 테이블이 계속 로딩 상태에서 멈춰 있습니다.

즉, 업비트만 `/api/premium/table-filtered`에서 정상적으로 데이터를 받고 있고,
빗썸/코인원 조합은 data가 비어있거나 에러가 나는 것 같습니다.

제가 보기에는 아래 둘 중 하나일 가능성이 큽니다:

1) 드롭다운 value와 API 내부 필터링값 불일치
   - 드롭다운: "BITHUMB_KRW" / "COINONE_KRW"
   - 내부 필터: exchange = "BITHUMB", quote = "KRW"
   - 이 매핑이 업비트만 구현되어 있고, 빗썸/코인원은 빠져있을 수 있습니다.

2) `exchange_markets.json`에서 빗썸/코인원 마켓이 제대로 매핑되지 않았거나,
   `/api/premium/table-filtered`에서 해당 거래소를 건너뛰는 로직이 있을 수 있습니다.

확인 부탁드리는 내용:

1. 브라우저에서 직접 호출:
   - `/api/premium/table-filtered?domestic=UPBIT_KRW&foreign=BINANCE_BTC`
   - `/api/premium/table-filtered?domestic=BITHUMB_KRW&foreign=BINANCE_BTC`
   - `/api/premium/table-filtered?domestic=COINONE_KRW&foreign=BINANCE_BTC`
   각각 호출했을 때 `data.length`가 어떻게 나오는지 확인 부탁드립니다.

2. 빗썸/코인원 조합에서 `data`가 빈 배열이라면,
   - `domestic` 파라미터를 어떻게 파싱해서
     `exchange` + `quote`로 나누는지 코드 블록을 한 번만 공유해 주세요.
   - 그리고 `exchange_markets.json`에서 빗썸/코인원 마켓이 어떤 구조로 들어 있는지도 같이 봐야 합니다.

지금 사용자 입장에서는 “기준 거래소”를 바꿔도 **업비트만 동작하고 나머지는 무한 로딩**이라
핵심 기능이 반쪽짜리인 상태라서, 이 부분을 최우선으로 잡아야 합니다.

=====================================
정리
=====================================
- 상단 차트: 기존 컨트롤은 살리고, 홈에서만 기본 심볼을 BINANCE:BTCUSDT로 고정
- 해외 드롭다운: 라벨을 마켓 기준으로 명확히 (BTC/USDT/선물 구분)
- 프리미엄 테이블: 빗썸/코인원 선택 시 무한 로딩 → `/api/premium/table-filtered`의 domestic 파싱/필터링 로직 점검

위 세 가지 수정 후, 
- 홈 상단 차트 스크린샷 1장
- 업비트/빗썸/코인원 각각 선택한 상태의 프리미엄 테이블 스크린샷
을 공유해 주시면, UI/UX 디테일과 컬럼 정렬은 ChatGPT 쪽에서 최종 정리하겠습니다.
