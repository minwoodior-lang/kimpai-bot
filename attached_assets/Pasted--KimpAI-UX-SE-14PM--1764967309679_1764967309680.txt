🧩 1. 상단 AI 요약 카드 모바일 최적화
문제

SE에서 텍스트가 카드 내부에서 줄바꿈되며 제목과 값이 겹침

카드 좌우 패딩이 과함

글자 크기가 PC 기준이라 작은 화면에 비해 비효율적

요구사항

모바일(sm 이하)에서 다음 적용:

padding: 14px → 10px
font-size: 15px → 13px
항목 간 세로 간격 6px → 3px
카드 전체 너비를 화면 양끝 100%로 확장
KR Premium Score bar를 더 짧게 (width 85% → 70%)


목표: 카드 높이를 최소화하고 SE 화면에서도 안정적으로 보이도록.

📊 2. 프리미엄 차트 영역 (드롭다운 + 개인화 설정)
문제

아이폰 SE에서 드롭다운 2개가 나란히 배치되면 width 초과 발생

두 개가 겹치거나 아래 텍스트가 내려감

“개인화 설정” 버튼 글자 길이 때문에 overflow 발생

요구사항
모바일(sm 이하):
드롭다운 2개를 가로 → 세로 스택으로 변경
gap 8px 적용
개인화 설정 버튼 font-size 14px → 12px
드롭다운 min-width 제거 (auto로 변경)


아이폰 SE에서 반드시 2줄 배치.

📈 3. TradingView 차트 위/아래 간격 조정
문제

SE에서 차트가 너무 큼 → 차트 아래 첫 번째 코인 행과 겹침

차트 위 컨트롤(1분/30분/1시간)이 두 줄로 내려가기도 함

요구사항

차트 높이 모바일 한정 300px → 240px 축소

차트 컨트롤 버튼 폰트 14px → 12px

버튼 padding 8px → 4px

📄 4. 코인 테이블(리스트) 모바일 최적화
문제

SE에서 “코인명 + 심볼 + 가격 + 김프 + 전일대비 + 거래대금” 6열은 과도함

일부 값이 겹쳐서 보이거나 투명도 중첩처럼 보임

행 높이가 과함

요구사항
(A) 모바일 전용 3열 레이아웃 제공 (SE 최적화)
1열: 코인아이콘 + 코인명/심볼
2열: 현재가 + 김프
3열: 전일대비 OR 거래대금 중 하나만 표시


우선순위: 거래대금 표시 > 전일대비 표시

(B) 텍스트 크기 축소
코인명: 14px → 12px  
심볼: 12px → 10px  
가격: 15px → 13px  
김프/전일대비: 14px → 12px  
거래대금: 13px → 11px

(C) 행 높이 줄이기
padding-y: 12px → 8px

🔽 5. 테이블 상단 통화/거래소 선택 드롭다운
문제

SE에서 업비트 KRW / 바이낸스 USDT 두 버튼이 폭을 초과함

오른쪽 버튼이 절단되거나 2줄로 밀림

요구사항

sm 이하에서 다음 적용:

두 버튼을 100% width로 배치 (세로 스택)
각 버튼 높이 36px → 32px 축소
글자크기 14px → 12px
좌우 padding: 12px → 6px

🔍 6. 검색창(Search bar)
문제

SE에서 placeholder 글자가 절단됨

양쪽 padding 과함

요구사항
padding-x: 14px → 8px
font-size: 14px → 12px
검색창 높이 44px → 38px

🦶 7. Footer (푸터)
문제

SE에서 2줄 문장이 3줄로 넘어가고 padding이 과함

공간 대비 글씨가 너무 큼

요구사항
font-size 13px → 11px
line-height 1.4 → 1.2
padding-y 24px → 14px
문단 최대폭 90%로 제한 (양끝 밀착 방지)

🛠 8. 전체 전역 스타일 조정

모바일(sm 이하)에서 공통 적용:

* { letter-spacing: -0.2px; }  // 글자 폭 감소로 줄바꿈 개선
body { font-size: 14px → 13px }
모든 버튼 radius 12px → 8px
컨테이너 좌우 padding 16px → 12px
