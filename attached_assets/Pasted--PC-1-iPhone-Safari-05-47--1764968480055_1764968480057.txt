1. 모바일 홈 우측 상단 메뉴(햄버거) 드롭다운 이상 표시

환경

iPhone Safari (스크린샷 05:47 기준)

현상

상단 우측 햄버거 메뉴 버튼을 눌렀을 때, 모바일 메뉴 드롭다운 레이아웃이 정상적으로 표시되지 않음.
(메뉴가 어색하게 표기되거나, 의도한 형태와 다르게 깨져 보임)

요청사항

모바일 뷰(md 미만)에서 햄버거 버튼 클릭 시:

화면 우측에서 슬라이드 인 되는 단일 메뉴 패널 형태로 깔끔하게 나오도록 정리.

메뉴 항목은 세로 리스트로 간격 일정하게, 텍스트는 왼쪽 정렬.

배경은 어두운 오버레이 + 메뉴 열려 있을 때 body 스크롤 잠금.

PC 뷰(md 이상)는 기존 상단 네비 그대로 사용하고, 모바일 전용 메뉴 레이아웃/로직을 분리해서 관리해달라.

2. 모바일 코인 테이블 컬럼 정렬(오름/내림차) 오류

환경

iPhone Safari

홈 화면 > 프리미엄 차트 아래 코인 테이블

재현 방법

모바일에서 코인 리스트까지 스크롤

테이블 헤더의 컬럼명(코인명, 현재가, 김프, 전일대비, 거래액(일) 등)을 탭해서 정렬 방향 변경 시도

현상

헤더를 눌렀을 때 정렬 화살표 아이콘은 바뀌지만, 실제 데이터 순서가 기대와 다름.

어떤 컬럼은 아예 반응이 없거나, 높은 값 순/낮은 값 순이 뒤섞여 보임.

특히 김프 / 전일대비 / 거래액(일) 같은 컬럼에서 정렬이 무작위처럼 느껴짐.

추정 원인

모바일용 컬럼 설정에서 sortKey가 잘못 매핑됐거나

화면에 표시하는 문자열(₩, +, %, 콤마 포함된 텍스트)을 그대로 정렬에 쓰고 있을 가능성 높음.
→ 문자열 기준 정렬이면 “+9.8%”, “+12.1%” 등이 제대로 숫자 순서로 안 맞을 수 있음.

수정 요청

PremiumTable.tsx의 정렬 로직을 전면 점검.

각 row에 정규화된 숫자 필드를 명시적으로 추가해서, 정렬은 항상 이 숫자를 기준으로만 수행:

priceNumeric : 현재가 (숫자)

premiumRate : 김프 퍼센트 (숫자)

changeRate24h : 전일대비 퍼센트 (숫자)

fromHighRate, fromLowRate 등 고가/저가 대비 퍼센트 (숫자)

UI에 표시하는 "₩4,530,000", "+1.63%" 같은 문자열은 표시용만 사용하고, 정렬에는 절대 사용하지 않게 분리.

모바일/PC 공통으로 쓸 수 있는 column key / sortKey 매핑을 하나로 통일:

예:

column.key = "change24h"

column.sortKey = "changeRate24h"

모바일에서 일부 컬럼이 숨겨져 있어도, 헤더 인덱스와 sortKey 매핑이 꼬이지 않도록

visibleColumns 배열 기준으로 클릭 이벤트 → 해당 column의 sortKey를 정확히 잡도록 로직 정리.

3. PC 코인 테이블 – 전일대비 / 고가대비 / 저가대비 정렬 뒤죽박죽

환경

PC 크롬

홈 화면 > 코인 테이블

재현 방법

PC에서 홈 접속

코인 테이블의 전일대비 / 고가대비 / 저가대비 헤더를 클릭하면서 오름/내림 정렬 전환

현상

퍼센트 기준으로 내림차순 / 오름차순 정렬이 되어야 하는데
실제로는 큰 값, 작은 값, 음수 값이 섞여 있어서 순서가 논리적이지 않게 보임.

일부 코인은 값이 없는 항목(null 또는 -)인데, 이 값들까지 섞여서 상단/중간에 나타남.

추정 원인

위와 같이 문자열 또는 잘못된 필드를 기준으로 정렬 중이거나,

전일대비/고가대비/저가대비 각각이 의도한 퍼센트 필드가 아니라 다른 값(절대 금액, null 포함)으로 정렬되고 있는 것으로 보임.

수정 요청

각 컬럼이 어떤 숫자 필드를 기준으로 정렬해야 하는지 명시적으로 매핑해달라:

전일대비 → changeRate24h (24h 퍼센트 변화율)

고가대비 → fromHighRate (금일 고가 대비 현재가 퍼센트)

저가대비 → fromLowRate (금일 저가 대비 현재가 퍼센트)

null / undefined / NaN 값은 항상 리스트 맨 아래로 보내는 규칙 적용.

가능한 경우 stable sort 사용해서 같은 값들 사이 순서가 계속 튀지 않도록.

4. 정렬 공통 규칙 통일 요청

모바일/PC 모두 아래 패턴으로 동작하게 통일 부탁드립니다.

초기 상태: 정렬 없음 (원본 순서)

컬럼 헤더 1회 클릭 → 내림차순 (가장 큰 값이 위)

2회 클릭 → 오름차순

3회 클릭 → 정렬 해제 (초기 상태로 복원)

정렬 상태(state)는 sortConfig = { key, direction } 형태로만 관리하고,
브레이크포인트에 따라 다른 정렬 로직이 돌지 않도록 공통화해 주세요.

이 세 가지가 지금 당장 체감되는 문제라,
모바일/PC 정렬 로직이랑 모바일 메뉴 드롭다운부터 먼저 안정화 부탁드립니다.
