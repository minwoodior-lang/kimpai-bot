🔥 6. Binance / Bybit 거래대금 정확도 문제 해결

현재 문제

Binance spot, Binance futures, Bybit spot, Bybit futures
거래대금이 전부 실제와 크게 다르게 표시됨
(예: 실제 3조 → 화면 100조)

🔥 반드시 해결할 핵심

Binance & Bybit는 절대 CoinGecko 글로벌 볼륨 사용하면 안 됨

거래소 API에서 제공하는 quoteVolume / turnover24h / volumeUSDT 등 “USDT 기준 거래대금”을 직접 사용해야 함

📌 올바른 필드
✔ Binance Spot

quoteVolume

/api/v3/ticker/24hr?symbol=BTCUSDT

✔ Binance Futures

volume × weightedAvgPrice 절대 금지

/fapi/v1/ticker/24hr 사용

quoteVolume 필드를 사용해야 함

✔ Bybit

turnover24h (USDT 기준)

📌 필수

프록시를 유지하더라도, Binance/Bybit는 실제 API 필드 사용해야 정확한 값 획득 가능.

🔥 7. "현재가" 표기 규칙 — 소수점 정확도

현재 문제

예: 73원이면 소수점 없이 73으로 표기됨

김프가에서는 71.50원처럼 원 단위도 세밀하게 표기

📌 요구 규칙
가격 범위	표기 방식
1,000원 이상	정수
100원 ~ 999원	소수점 1자리
1원 ~ 99원	소수점 2자리
1원 미만	소수점 4자리

예시

73원 → 73.00원

0.9원 → 0.9000원

2,340원 → 2,340원

🔥 10. 마켓 변경 시 차트 연동 (미상장 포함)

현재 문제

미상장 코인은 차트 버튼이 사라짐

차트는 Binance 또는 선택된 해외 거래소 기준으로 항상 표시 가능해야 함

📌 요구 사항

미상장 코인이라도 차트는 항상 inject

차트 우측 상단에는 마켓명 표시 (예: Binance / Bybit / OKX 등)

📌 마지막 정리 — 이번 전달문에 포함된 전체 태스크
1. 국내 마켓 구조 정상화 (KRW/BTC/USDT 분리)
2. 모든 거래소 심볼(파비콘) 적용
3. 코인 아이콘 적용
4. 미상장 코인 처리 (삭제 금지, "-" 처리, 차트 유지)
5. 거래대금 정렬 기본값 = 큰 순
6. Binance / Bybit 거래대금 정확도 복구 (프록시+실API quoteVolume)
7. 현재가 소수점 규칙 적용
8. 즐겨찾기(★) UI 적용
9. 모바일 테이블 5컬럼 구조 재구성  
   - 국내가 위 / 해외가 아래  
   - 스크롤 없이 다 보이도록
10. 미상장 코인도 차트버튼 유지
