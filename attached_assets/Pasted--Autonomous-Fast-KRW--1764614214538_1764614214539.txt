⚠️ 중요: Autonomous / 자율 모드는 절대 켜지 마.
Fast 모드에서만 작업해줘.

목표:
업비트 KRW 마켓 기준으로 코인 심볼 리스트를 가져온 다음,
각 심볼에 대해 CoinGecko에서 아이콘을 찾아서

- /public/coins/{SYMBOL}.png 에 저장하고
- /data/symbolIcons.json 파일에 "SYMBOL": "/coins/SYMBOL.png" 매핑을 자동으로 추가하는
Node 스크립트(fetchIcons.js)를 만들어줘.

이미 CoinIcon.tsx는 /coins/{SYMBOL}.png 를 아이콘 1순위로 사용하도록 수정된 상태이기 때문에,
이번 작업에서는 CoinIcon.tsx는 절대 수정하지 말고,
fetchIcons.js 파일만 새로 만들어줘.

상세 요구사항:

1) 파일 위치 및 환경
--------------------------------
- 프로젝트 루트(Next.js 기준) 위치에 fetchIcons.js 파일을 새로 만들어줘.
- Node 환경에서 실행 가능한 일반 JS 스크립트면 된다. (타입스크립트 불필요)
- Node 18 이상이면 글로벌 fetch를 사용하고,
  만약 Replit 환경이 fetch를 지원하지 않으면 node-fetch를 사용해도 된다.
  (이 경우 필요한 의존성 설치 주석을 코드 상단에 남겨줘)

2) 업비트에서 KRW 마켓 심볼 가져오기
---------------------------------------
- Upbit API: https://api.upbit.com/v1/market/all 를 호출해서 전체 마켓 리스트를 가져와라.
- 그 중에서 market 이 "KRW-" 로 시작하는 항목만 필터링해서,
  "KRW-BTC" → "BTC" 처럼 심볼 리스트를 만든다.
- 대소문자는 전부 대문자 SYMBOL 기준으로 사용 (BTC, ETH, SUI, …).

3) /public/coins 와 /data/symbolIcons.json 준비
------------------------------------------------
- public/coins 폴더가 없으면 생성한다.
- /data/symbolIcons.json 파일이 있으면 기존 내용을 읽어서 JS 객체로 파싱한다.
  - 없으면 빈 객체 {} 로 시작한다.
- 이미 /public/coins/SYMBOL.png 파일이 존재하는 심볼은
  새로 다운로드하지 않고 건너뛰고,
  symbolIcons.json 에 "SYMBOL": "/coins/SYMBOL.png" 매핑이 없다면 추가만 해준다.

4) CoinGecko에서 아이콘 검색 + 다운로드
-----------------------------------------
- 각 심볼에 대해 CoinGecko Search API
  https://api.coingecko.com/api/v3/search?query={symbol}
  를 호출해서 후보들을 가져온다.
- 반환된 coins 배열 중에서:
  - 우선 symbol이 정확히 일치하는 후보를 우선 선택하고,
  - 필요한 경우 업비트 english_name / korean_name 과 이름이 제일 비슷한 코인을 선택하는 식으로
    어느 정도 합리적인 매칭을 시도한다.
- 선택한 코인의 image.large (또는 image.thumb) URL을 사용해서
  아이콘 이미지를 다운로드하고,
  /public/coins/{SYMBOL}.png 로 저장해라.
- 다운로드가 성공하면 symbolIcons["SYMBOL"] = "/coins/SYMBOL.png" 로 설정한다.
- CoinGecko에 결과가 없거나 이미지 URL이 없어서 실패하는 심볼은
  별도로 failedSymbols 배열에 모아두고, 마지막에 콘솔에 출력한다.

5) rate limit / 안전장치
--------------------------
- CoinGecko 요청 사이에는 1~1.5초 정도 sleep을 두어서
  rate limit(429) 에 걸리지 않도록 한다. (setTimeout + await sleep 함수 사용)
- 네트워크 에러나 단일 코인 처리 실패는 전체 스크립트를 중단하지 말고,
  해당 심볼만 실패 목록에 추가한 뒤 다음 심볼을 계속 처리하도록 해줘.

6) symbolIcons.json 저장
--------------------------
- 모든 심볼 처리 후에는 symbolIcons 객체를
  /data/symbolIcons.json 에 JSON.stringify(map, null, 2) 형태로 저장한다.
- 키(SYMBOL)는 알파벳 순으로 정렬한 뒤 저장하면 관리에 좋다.
- 마지막에 콘솔에 다음과 같이 요약 로그를 출력하도록 해줘:
  - 총 시도한 심볼 수
  - 성공 개수 / 실패 개수
  - 실패 심볼 목록 (있다면)

7) 실행 방법 안내 주석
------------------------
- fetchIcons.js 상단 또는 하단에 주석으로
  "Replit Shell에서 `node fetchIcons.js` 로 실행하면 됩니다."
  라는 안내를 남겨줘.

중요:
- 이번 작업에서는 CoinIcon.tsx, PremiumTable.tsx, 다른 컴포넌트/페이지 코드는 수정하지 말고,
  오직 fetchIcons.js 파일과 /public/coins, /data/symbolIcons.json 관련 로직만 추가/사용해줘.
