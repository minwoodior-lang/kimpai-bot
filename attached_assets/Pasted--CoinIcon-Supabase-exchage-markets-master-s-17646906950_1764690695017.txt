지금 이름 표기 로직이랑 CoinIcon 쪽 정리는 감사드립니다.

다만 현재 상태가 이렇습니다:

- Supabase 테이블(exchage_markets, master_symbols)에는 name_ko가 전부 잘 들어가 있음
- 테이블 데이터 조회 화면에서도 모두 한글로 표기되는 것 확인
- 그런데 실제 /api/premium/table 응답이랑 프론트에서 일부 코인은 여전히 영문으로 표시되고,
  브라우저 콘솔에 런타임 에러가 발생 중입니다.

즉, DB → API → 프론트 사이에서 필드 매핑이 어딘가 끊겨 있는 것 같습니다.

아래 3가지 확인/수정을 부탁드립니다.

────────────────────
1) /api/premium/table 응답 구조 먼저 확인
────────────────────
- /api/premium/table.ts에서 Supabase 쿼리 결과를 그대로 console.log 하거나,
  첫 1~2개 row를 샘플로 찍어서 아래 필드들이 실제로 내려오는지 확인 부탁드립니다.

  기대하는 필드:
    base_symbol
    name_ko
    name_en
    icon_url (있으면 사용, 없으면 프론트에서 base_symbol 기준으로 아이콘 매칭)

- 현재 PremiumTable.tsx에서는 PremiumData 인터페이스에
  name_ko, name_en, icon_url을 이미 추가한 상태이므로,
  API 응답 JSON에도 동일한 키 이름으로 들어와야 합니다.

  만약 API 응답에서 다른 이름(예: ko_name, en_name, metadata.name 등)을 쓰고 있다면,
  /api/premium/table.ts에서 key를 맞춰 주시거나, 프론트 매핑을 조정해 주세요.

────────────────────
2) /api/premium/table.ts에서 name_ko / name_en SELECT/매핑 추가
────────────────────
- Supabase 쿼리에서 exchange_markets / master_symbols 조인 시,
  name_ko와 name_en 컬럼이 실제로 SELECT 되고 있는지 확인 부탁드립니다.

  예시:
    select(`
      ...,
      base_symbol,
      name_ko,
      name_en,
      icon_url
    `)

- 그리고 최종 JSON으로 리턴할 때도
  row.name_ko, row.name_en, row.icon_url 을 그대로 넣어줘야
  PremiumTable.tsx의 getDisplayName(name_ko ?? name_en ?? base_symbol) 로직이 제대로 동작합니다.

────────────────────
3) 브라우저 런타임 에러 원인 잡기
────────────────────
- 지금 콘솔에 찍히는 런타임 에러 메시지를 기준으로,
  PremiumTable.tsx에서 undefined 필드를 접근하는 부분이 있는지 확인 부탁드립니다.

  예:
    - data.name_ko가 undefined 인데 바로 toLowerCase() 같은 메서드를 호출하는 경우
    - icon_url이 없는 상태인데 문자열로 가정하고 사용하는 경우

- 가능하면 /api/premium/table 응답 샘플 1건을 기준으로
  PremiumData 타입과 실제 응답 구조가 1:1로 맞는지 다시 한 번만 검증해 주세요.

요약하면:

- DB에는 한글명이 다 들어가 있고
- 프론트 로직도 name_ko ?? name_en ?? base_symbol 으로 통일된 상태인데,
- /api/premium/table 응답에서 name_ko / name_en / icon_url이 제대로 매핑되지 않아서
  일부 코인이 영문으로 보이고, 런타임 에러가 나는 것 같습니다.

API 응답 구조를 실제로 확인하고,
select / 매핑 / 타입을 한 번만 정리해 주시면 이 문제가 해결될 것 같습니다.
