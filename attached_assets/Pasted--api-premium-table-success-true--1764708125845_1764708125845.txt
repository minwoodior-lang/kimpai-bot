방금 보내주신 내용 잘 확인했습니다.

- 백엔드 시세 수집 및 프리미엄 계산 정상 동작
- /api/premium/table 응답 구조 및 success: true 필드 추가
- 프론트와의 필드 매핑도 정리해 주셔서 감사합니다.

이제 다음 단계로, 아래 작업들을 순차적으로 진행 부탁드립니다.

────────────────────────
1) 국내 3개 거래소 전체 상장 코인 자동 동기화
────────────────────────

목표: 업비트 / 빗썸 / 코인원에 코인이 새로 상장되어도
제가 JSON을 수동 수정하지 않고 자동으로 반영되도록 만드는 것입니다.

1-1. 업비트 (Upbit)
- API: GET https://api.upbit.com/v1/market/all?isDetails=true
- 이 엔드포인트에서 KRW/BTC/USDT 마켓 전체를 가져와서
  exchange_markets.json 에 반영해 주세요.

예)
  "market": "KRW-BTC",
  "korean_name": "비트코인",
  "english_name": "Bitcoin"

→ 이 값을 이용해서:
  - exchange: "UPBIT"
  - base_symbol: "BTC"
  - quote_symbol: "KRW"
  - name_ko, name_en 도 같이 세팅

1-2. 빗썸 (Bithumb)
- API 예시:
  - https://api.bithumb.com/public/ticker/ALL_KRW
  - https://api.bithumb.com/public/ticker/ALL_BTC
  - https://api.bithumb.com/public/ticker/ALL_USDT

- 최상위 key가 심볼(예: "BTC", "ETH", "XRP") 이므로,
  이를 base_symbol 로 사용하고
  각각 KRW/BTC/USDT 마켓으로 exchange_markets.json 에 반영해 주세요.

예)
  {
    "exchange": "BITHUMB",
    "market_symbol": "KRW-BTC",
    "base_symbol": "BTC",
    "quote_symbol": "KRW"
  }

1-3. 코인원 (Coinone)
- API 예시:
  - https://api.coinone.co.kr/ticker?currency=all

- 응답에서 각 코인의 통화(symbol)를 base_symbol 로 보고
  KRW 마켓으로 exchange_markets.json 에 반영해 주세요.

예)
  {
    "exchange": "COINONE",
    "market_symbol": "KRW-BTC",
    "base_symbol": "BTC",
    "quote_symbol": "KRW"
  }

1-4. 자동화 방식
- 위 세 거래소의 마켓/심볼 정보를 기반으로
  exchange_markets.json 을 **스크립트에서 재생성**하는 구조로 만들어 주시면 좋겠습니다.
- 새 상장이 생기면, 이 스크립트만 돌려도
  자동으로 exchange_markets.json 이 최신 상태가 되도록 해 주세요.

────────────────────────
2) master_symbols.json 메타데이터 자동 보강
────────────────────────

목표: base_symbol 기준으로 name_ko / name_en / icon_url 이 최대한 자동으로 채워지도록.

- 새로 발견된 base_symbol 이 master_symbols.json 에 없으면:
  1) 업비트 market/all 응답의 korean_name / english_name 우선 사용
  2) 빗썸/코인원에서도 한글/영문명이 제공되면 보조로 사용
  3) 그래도 없으면:
     - name_en 정도만 외부 메타데이터(CoinGecko/CoinMarketCap 등)에서 가져오는 구조 고려
  4) name_ko 가 없으면 null 허용 (나중에 제가 수동 보정할 수 있도록)

- icon_url 은 현재 /coins/심볼.png 또는 기존 규칙을 그대로 유지해 주시되,
  심볼이 새로 추가되면 기본 placeholder 아이콘이라도 매핑되게 해 주세요.

결과적으로:
- master_symbols.json 은 base_symbol 기준으로
  국내 거래소에 상장된 코인을 전부 포함하도록 유지해 주시면 됩니다.

────────────────────────
3) premiumTable.json 코인 확장
────────────────────────

현재는 6개 코인(BTC, ETH, XRP, ADA, DOGE, SOL) 기준으로 동작하고 있는데,
위에서 확장된 exchange_markets.json / master_symbols.json 을 기반으로
더 많은 코인을 지원하는 방향으로 확장 부탁드립니다.

- 로직:
  1) exchange_markets.json 에서 지원 대상 심볼 목록 추출
  2) 국내 시세(업비트 우선, 필요시 빗썸/코인원 보조) + 해외 USDT 시세(Binance/OKX/Bybit 등)를 가져와서
  3) premiumTable.json 에 base_symbol 단위로 프리미엄 계산 결과 저장

- 성능/타임아웃은 배치 처리/딜레이 등으로 조절해 주셔도 됩니다.
  (한 번에 너무 많은 심볼을 호출하지 않도록 분할 처리)

────────────────────────
4) 검증용 로그/체크
────────────────────────

작업 완료 후에는, 아래를 확인해 주시면 좋겠습니다.

1) premiumTable.json 내 base_symbol 개수
2) exchange_markets.json / master_symbols.json / premiumTable.json 간
   base_symbol 기준 커버리지 비교 결과
   (누락된 심볼이 없도록)

이 과정에서 사용하신 체크 스크립트나 로그가 있다면
함께 공유해 주시면, 이후 단계(탭/필터/정렬/AI 분석 탭 등) 설계에 참고하겠습니다.
