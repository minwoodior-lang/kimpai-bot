🔹 홈 테이블 UX 관련 재확인 + 수정 요청

레포트 주신 내용대로라면

해외/국내 현재가 깜빡임 복구

코인셀/숫자셀 폰트 통일

이 모두 적용됐다고 하셨는데, 실제 라이브 화면에서는 아직 전혀 반영되지 않은 상태입니다.
현재 확인된 문제와, 해야 할 작업을 다시 정리해서 요청드립니다.

1. 현재가 깜빡임(PriceCell) 동작 재점검 요청

현재 상태

홈 테이블 현재가 컬럼에서

국내 현재가(상단)도

해외 현재가(하단)도
가격이 바뀌어도 깜빡임 애니메이션이 전혀 발생하지 않습니다.

말씀해주신 TwoLinePriceCell 이 실제 테이블에 적용되지 않았거나,
이전 PriceCell의 플래시 로직이 끊긴 상태로 보입니다.

원하는 동작

현재가 셀은 국내/해외 가격 둘 중 하나라도 값이 변하면
이전 값과 비교해서:

상승 → price-flash-up

하락 → price-flash-down
클래스를 셀 전체에 적용해 주세요.

애니메이션 후에는:

상단 국내 가격은 기존 색상(흰색)으로 복귀

하단 해외 가격은 회색(text-gray-500)으로 복귀

기존에 구현되어 있던 배경/텍스트 플래시 로직 자체는 그대로 사용하되,
단지 “기본 텍스트 색상만 회색으로 바꾼 것”이어야 합니다.

구체 요청

PremiumTable.tsx에서 현재가 컬럼이 실제로 TwoLinePriceCell(또는 PriceCell)을 사용하고 있는지 확인해 주세요.

TwoLinePriceCell 내부에서 이전 값과 현재 값을 비교하는 로직이
상단 값만 보고 있는지, 아니면 상·하단 둘 다를 체크하는지 다시 한번 확인해 주세요.

가격이 변하는 상황에서 className에 price-flash-up / price-flash-down이 들어오는지 DevTools로 직접 확인 후,
실제 적용이 안 되고 있다면 해당 부분을 수정해 주세요.

2. 코인셀(코인명) 폰트 스타일 통일 요청

현재 상태

코인명 셀에서:

한글 이름 / 심볼(또는 영문명)의 폰트 크기와 굵기가 행마다 들쭉날쭉합니다.

레포트에 적어주신 text-sm font-medium / text-xs text-gray-500 통일이 실제 화면에는 적용되어 있지 않습니다.

원하는 구조

// 코인명 셀 공통 스타일 예시
<div className="flex flex-col">
  <span className="text-sm font-medium">
    {/* name_ko (한글 코인명) */}
  </span>
  <span className="text-xs text-gray-500">
    {/* symbol 또는 name_en */}
  </span>
</div>


모든 코인행에서 위와 같은 동일한 className을 사용해 주세요.

코인명 관련 컴포넌트가 따로 있다면, 그 안에 이 스타일을 고정해 주시면 됩니다.

3. 모든 숫자 셀(TwoLineCell) 폰트/정렬 완전 통일 요청

현재 상태

현재가 / 김프 / 전일대비 / 고가대비 / 저가대비 / 거래액(일) 각 컬럼의
상단/하단 숫자 텍스트 크기와 굵기가 컬럼마다 조금씩 다릅니다.

보고서에 쓰신 “전역 통일”과 실제 UI가 일치하지 않습니다.

원하는 공통 스타일

// 모든 숫자 셀 공통 TwoLineCell 예시
<div className="flex flex-col items-end leading-tight">
  <span className="text-sm font-medium">
    {/* 상단 값 (국내 기준 메인 값 / % 값) */}
  </span>
  <span className="text-xs text-gray-500">
    {/* 하단 값 (해외 기준 보조 값 또는 '-') */}
  </span>
</div>


상단: 전부 text-sm font-medium
(현재가, 김프%, 전일대비%, 고가대비%, 저가대비%, 거래액(일) 상단 포함)

하단: 전부 text-xs text-gray-500

정렬: items-end + text-right 로 상·하단 모두 오른쪽 정렬

줄 간격: leading-tight 또는 일정 gap 으로 고정

구체 요청

TwoLineCell 컴포넌트가 있다면, 해당 컴포넌트 안에서
위 className을 하드코딩 수준으로 고정해서,
각 컬럼에서 별도로 다른 스타일을 덮어쓰지 않도록 해 주세요.

수정 후, DOM 상에서 모든 숫자 셀의 상단이 text-sm font-medium,
하단이 text-xs text-gray-500로 동일하게 찍히는지 한 번만 확인해 주세요.

정리하면,

현재가 컬럼 깜빡임(PriceCell / TwoLinePriceCell) 실제 동작 복구

코인명 셀 폰트 구조 통일

모든 숫자 셀(TwoLineCell) 폰트·정렬 완전 통일

이 세 가지가 아직 라이브 화면에 반영되지 않았습니다.
위 내용 기준으로 다시 한 번 점검 및 수정 부탁드립니다.
