🔧 KimpAI 홈 화면 전면 수정 요청 (KIMPGA 스타일 기준)
0. 목표 요약

참고 사이트: KIMPGA 홈

KimpAI 홈도 아래 구조로 맞추고 싶습니다.

상단: 단일 TradingView 차트 (BTC/USDT – Binance 기준)

하단:

기준 거래소(국내) 드롭다운

해외 거래소 드롭다운

선택된 조합에 맞는 프리미엄 테이블

각 국내 거래소 기준으로 실제 상장 코인 전체가 표기되어야 하고,
해외 거래소 기본값은 Binance BTC 마켓이어야 합니다.

리스트는 기본적으로 거래대금(24h) 기준 내림차순 정렬이 되면 좋겠습니다.

A. 상단 TradingView 차트 수정

현재 홈 상단 차트가 Apple 종목처럼 전혀 다른 차트가 나오거나,
차트가 두 개 겹쳐서 표시되는 문제가 있습니다.

원하는 동작:

차트는 단일 1개만 표시

기본 심볼: BINANCE:BTCUSDT (또는 Binance BTC 마켓에서 사용하는 BTC/USDT 심볼)

타임프레임 버튼: 1분, 5분, 15분, 30분, 1시간 등 현재 구조 유지

홈 하단에 있는 국내/해외 거래소 선택과 완전히 분리된 독립 차트여도 괜찮습니다.
(일단은 “기본 BTC/Binance 차트 1개”만 정확하게 나오도록 해 주세요.)

요청사항 A

홈 상단 TradingView 컴포넌트를:

하나만 렌더링되도록 정리

기본 심볼을 BTC/USDT (Binance)로 고정 설정

차트 관련 props/심볼 설정 코드 블록을 보여주시면,
필요하면 ChatGPT 쪽에서 심볼명을 정확히 다시 지정해 드리겠습니다.

B. 국내/해외 거래소 드롭다운 동작 정의
1) 기준 거래소(국내) 드롭다운

드롭다운 항목:

업비트 KRW

빗썸 KRW

코인원 KRW

기본 선택값: 업비트 KRW

이 값에 따라 프리미엄 테이블에 등장하는 코인 목록이 달라져야 합니다.

예: “업비트 KRW” 선택 시 → 업비트 KRW 마켓에 상장된 코인 전체

“빗썸 KRW” 선택 시 → 빗썸 KRW 상장 코인 전체

“코인원 KRW” 선택 시 → 코인원 KRW 상장 코인 전체

2) 해외 거래소 드롭다운

드롭다운 항목(예시):

바이낸스 BTC 마켓

바이낸스 USDT 마켓

OKX USDT

Bybit USDT

Bitget USDT

Gate USDT

HTX USDT

MEXC USDT

기본 선택값: 바이낸스 BTC 마켓

이 값은 “프리미엄 계산 시 어떤 해외 시세를 기준으로 할지”를 의미합니다.
(실제 프리미엄 계산 로직은 이미 구현되어 있다면, 선택된 해외 거래소에 맞추어 호출/필터만 바꿔주시면 됩니다.)

요청사항 B

홈 화면에서 사용하는 API/쿼리 레벨에서:

domesticExchange (UPBIT/BITHUMB/COINONE)

foreignExchange (BINANCE/OKX/BYBIT/...)
를 기준으로 필터링할 수 있도록 구조 정리 부탁드립니다.

드롭다운의 현재 선택값을 API 요청 파라미터나 프론트 필터로 연결할 수 있게 해 주세요.

C. 프리미엄 테이블(코인 리스트) 구성 및 정렬
1) 표시 대상

선택된 “기준 거래소(국내)”에 실제 상장된 코인 전체가 한 번에 리스트업 되어야 합니다.
(참고: KIMPGA처럼 업비트 KRW/빗썸 KRW 선택 시, 각 거래소의 상장 코인 전부 출력)

각 행은 “한 코인(심볼)”을 기준으로 정리되면 됩니다.

예: BTC, ETH, SOL, SUI, XRP …

가능하다면 아래 구조를 추천합니다:

{
  symbol: "BTC",
  name_ko: "비트코인",
  name_en: "Bitcoin",
  market: "BTC/KRW",              // 기준 거래소의 마켓
  domesticExchange: "UPBIT",      // 또는 BITHUMB / COINONE
  foreignExchange: "BINANCE",     // 현재 선택된 해외 거래소
  koreanPrice: number,            // 국내 현재가 (KRW)
  globalPrice: number,            // 해외 현재가 (USD 또는 KRW 환산)
  globalPriceKrw: number,
  premium: number,                // 김프 %
  change24h: number,              // 전일 대비 %
  high24h: number,
  low24h: number,
  volume24hKrw: number,           // 24h 거래대금 (KRW 기준)
  iconUrl: string | null
}

2) 테이블 컬럼 (KIMPGA 스타일 참고)

코인명 (한글명 + 심볼, 아이콘 포함)

현재가 (국내 기준, KRW)

김프 (%)

전일대비 (%)

고가 / 저가 (24h 또는 52주, 현재 구현된 기준 따라)

거래대금 (24h, “만 원” 단위 표기 가능)

3) 기본 정렬

기본 정렬 기준: volume24hKrw 내림차순 (거래대금 많은 순)
→ KIMPGA처럼 유동성 높은 코인이 위로 오게.

추가로, 테이블 헤더 클릭 시:

김프 기준 정렬

전일대비 기준 정렬
등을 할 수 있으면 좋지만, 필수는 아닙니다.
최소한 기본 정렬만 거래대금 내림차순으로 맞춰 주세요.

4) 기타 요구사항

리스트 상단에 “암호화폐 총 N개” 텍스트가 KIMPGA처럼 표시되면 좋겠습니다.

N = 현재 테이블에 노출된 코인 수

현재 KimpAI 홈에서는 이 영역이 무한 로딩 상태로만 남아 있는 문제가 있습니다.

API 에러거나 빈 배열일 경우에는:

무한 로딩 대신 “데이터를 불러오지 못했습니다” 등의 문구가 뜨도록 처리 부탁드립니다.

요청사항 C

홈 화면 전용 API 또는 기존 API를 정리해서:

선택된 국내/해외 거래소 조합에 따라 위와 같은 구조의 리스트를 반환

정렬은 기본 거래대금 내림차순

이 API 응답 예시(JSON 2~3개 정도)를 공유해 주시면,
ChatGPT 쪽에서 프론트 테이블 렌더링/정렬에 맞는 추가 지시를 드리겠습니다.

D. 아이콘 & 한글명 표기 (홈 테이블 기준)

홈 테이블에서는 반드시 한글명 + 심볼 + 아이콘이 함께 보여야 합니다.

예: “비트코인 (BTC)” 왼쪽에 원형 아이콘

master_symbols 또는 symbolIcons.json에서:

symbol 기준으로 ko_name, icon_url을 가져와서 매핑해 주세요.

만약 market이 BTC/KRW 형식이라면, market.split("/")?.[0] 으로 base 심볼만 따서 매칭하는 방식이면 됩니다.

요청사항 D

홈 페이지에서 사용하는 코인 리스트 데이터에
symbol, name_ko, name_en, iconUrl 필드를 포함해 주세요.

CoinIcon 컴포넌트가 있다면, 이 필드를 기준으로 아이콘이 안정적으로 뜨도록 확인 부탁드립니다.
