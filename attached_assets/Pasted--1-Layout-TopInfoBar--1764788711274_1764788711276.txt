🔌 3. 데이터 연동 방식 (전부 내부 API 기준)

모든 숫자는 클라이언트에서 외부 API 직접 호출하지 말고,
Replit 내부 서버(Next.js API)에서 한 번에 내려주는 /api/global-metrics 로 묶어주세요.

3-1. /api/global-metrics 응답 형태 예시
type GlobalMetricsResponse = {
  usdKrw: number;          // 환율
  usdKrwChange: number;    // 환율 변화율 (%)

  tetherKrw: number;       // 테더 KRW 시세
  tetherChange: number;    // 테더 변화율 (%)

  btcDominance: number;    // BTC 점유율 (%)
  marketCapKrw: number;    // 시가총액(원 단위)
  marketCapChange: number; // 시총 변화율 (%)

  volume24hKrw: number;    // 24시간 거래량(원 단위)
  volume24hChange: number; // 거래량 변화율 (%)

  concurrentUsers: number; // 현재 동시 접속자 수
};


프론트 상단 인포바 컴포넌트는 이 API만 한 번 호출해서 위 값들 바인딩하면 됩니다.

3-2. 각 값은 어디서 가져오는지

환율(usdKrw / usdKrwChange)

Replit 서버에서 FX API 1곳(예: exchangerate.host 등)만 사용

서버 사이드에서 5~10분 캐시 (메모리 or json) 후 /api/global-metrics 에서 반환

테더(tetherKrw / tetherChange)

빗썸 USDT/KRW 시세 기준으로 가져와 주세요.

이미 국내 거래소 시세 수집 로직이 있다면 거기서 재사용 가능하고,
없다면 서버에서 빗썸 public ticker 한 번 호출 → 캐시 후 반환.

BTC 점유율 / 시총 / 24시간 거래량

지금 premium 계산에 사용하는 글로벌 시세 소스(CoinGecko/CMC 등)와 동일한 곳 사용

마찬가지로 서버에서 주기적으로 업데이트해서 /api/global-metrics에 합쳐서 내려주기.

동시접속자(concurrentUsers)

현재 구현되어 있는 내부 로직(웹소켓/redis/메모리 카운터 등) 값 그대로 넣어주면 됩니다.

없다면 초기에 임시 더미 값에서 시작해도 되는데, 인터페이스만 맞춰두면 나중에 실제 값으로 교체 가능.

📩 레플릿에게 그대로 보내면 되는 메시지 요약

헤더 위에 임시로 올려둔 텍스트 설명은 전부 삭제해 주세요.

상단 인포바는 지금 구조 유지하되, 왼쪽에 환율(구글 파비콘), 테더 USDT(빗썸 USDT 아이콘)를 배치하고,
그 다음에 BTC 점유율, 시가총액, 24시간 거래량, 우측 끝에 동시접속자가 오도록 정리해 주세요.

라벨은 BTC → BTC 점유율, 거래량 → 24시간 거래량 으로 수정해 주세요.

모든 숫자는 /api/global-metrics 한 곳에서 가져오도록 만들고, 이 API는 Replit 내부 서버에서
(1) FX API로 받은 USD/KRW, (2) 빗썸 USDT/KRW, (3) 글로벌 시총/거래량/BTC Dominance,
(4) 동시접속자 수를 모아서 { usdKrw, tetherKrw, btcDominance, marketCapKrw, volume24hKrw, ... } 형식으로 반환하게 구현해 주세요.

클라이언트는 외부 API를 직접 호출하지 않고, /api/global-metrics만 호출해서 상단 인포바를 렌더링하도록 수정해 주세요.
