지금까지 아이콘 파이프라인 구조는 잘 정리된 것 같습니다.

- buildPremiumTable: master_symbols.icon_path → icon_url 매핑 ✅
- /api/premium/table-filtered: exchange:symbol 기준 icon_url 적용 ✅
- CoinIcon: icon_url → 로컬 → CDN → 회색 동그라미 ✅
- 잘못되었던 "UPBIT 아이콘이 다른 거래소에도 같이 쓰이던 문제"도 정리하신 것 확인했습니다.

다만, 실제 화면 기준으로 보면:

- 여전히 몇몇 코인은 "잘못된 로고"가 붙어있는 상태입니다.
- 예: 볼타(A), 휴머니티(H) 등은 업비트/빗썸/코인원 공통으로
  실제 해당 프로젝트 로고가 아니라 다른 코인(솔라나/이더리움 등) 아이콘처럼 보입니다.

지금부터는 아이콘 수집/부족 문제를 떠나서,
**“이 코인들은 아이콘을 완전히 쓰지 말고, 항상 이니셜 Placeholder로만 보여 달라”**
라는 예외 처리가 필요합니다.

===============================
1) 예외 심볼 리스트 만들기
===============================
국내 거래소 공통으로 "아이콘을 강제로 끄고 싶은" 심볼들을 배열로 관리해 주세요.

예시:

```ts
// src/config/badIconSymbols.ts (예시)
export const BAD_ICON_SYMBOLS = [
  "H",  // Humanity Protocol
  "A",  // Volta
  // 앞으로 확인되는 심볼들 추가 예정
];
===============================
2) API 또는 CoinIcon에서 예외 처리
아래 둘 중 편한 방식으로 적용해주시면 됩니다.

A안) API 레벨에서 icon_url 강제 null 처리

/api/premium/table-filtered.ts 에서 응답 만들 때,

ts
코드 복사
import { BAD_ICON_SYMBOLS } from "@/config/badIconSymbols";

const shouldForcePlaceholder = BAD_ICON_SYMBOLS.includes(symbol.toUpperCase());

return {
  ...entry,
  icon_url: shouldForcePlaceholder ? null : entry.icon_url,
};
이렇게 하면 BAD_ICON_SYMBOLS 에 들어있는 심볼들은
어떤 거래소이든 무조건 icon_url = null 로 내려가게 되고,
CoinIcon 에서 항상 회색 동그라미 / 이니셜 Placeholder로만 보이게 됩니다.

B안) CoinIcon 레벨에서 바로 차단

CoinIcon 컴포넌트에서:

tsx
코드 복사
import { BAD_ICON_SYMBOLS } from "@/config/badIconSymbols";

const upperSymbol = symbol?.toUpperCase();
const forcePlaceholder = upperSymbol && BAD_ICON_SYMBOLS.includes(upperSymbol);

const finalIconUrl = forcePlaceholder ? null : iconUrl;

// finalIconUrl 이 null 이면 동그라미 + 이니셜 렌더링
이렇게 하면 BAD_ICON_SYMBOLS 에 포함된 심볼들은
icon_url 이 무엇이든 간에 UI에서는 항상 이니셜 Placeholder만 사용합니다.

===============================
3) 이니셜 Placeholder 스타일
아이콘이 null 이거나 BAD_ICON_SYMBOLS 에 속하면,
아래처럼 이니셜 동그라미로 통일해 주세요.

원형 배경 (회색/어두운 톤)

중앙에 symbol 첫 글자 (대문자)

예: H, A, X …

대략 이런 느낌:

tsx
코드 복사
function PlaceholderIcon({ symbol }: { symbol: string }) {
  const initial = symbol?.[0]?.toUpperCase() ?? "?";

  return (
    <div className="flex h-6 w-6 items-center justify-center rounded-full bg-neutral-700 text-xs font-semibold text-white">
      {initial}
    </div>
  );
}
이 Placeholder는

icon_url 이 null 이거나,

BAD_ICON_SYMBOLS 에 포함된 심볼일 때만 사용해 주세요.

===============================
정리
지금 구조는 "아이콘이 없는(null) 코인 → 회색 동그라미" 까지는 잘 되어 있습니다.

다만 실제로는, 잘못된 로고가 붙어 있는 코인들이 몇 개 있어서,
이 코인들은 아이콘을 아예 끄고 이니셜 Placeholder만 쓰는 예외 처리가 필요합니다.

BAD_ICON_SYMBOLS 배열을 두고,
API 레벨 또는 CoinIcon 레벨에서 icon_url 을 강제로 null 처리하는 방식으로
업비트/빗썸/코인원 공통으로 적용 부탁드립니다.

이렇게 적용되면:

정상 매핑된 코인 → 원래 로고 정상 출력

아이콘 못 구한 코인 → 회색 이니셜 Placeholder

"잘못된 아이콘"이었던 코인(H, A 등) → 이니셜 Placeholder로 강제 전환

이 상태면 아이콘 신뢰도 문제는 한 번에 정리될 것 같습니다.
